# ContractHub TUI Design Specification (Extensible & Dynamic)

## Overview
This specification outlines the design and implementation of a new extensible Terminal User Interface (TUI) for `contracthub`. The primary goal is to provide a k9s-like interactive interface that not only works for the standard `contracthub` commands (excluding `enrich`), but is also built dynamically.

By designing it dynamically, when other teams import `contracthub` as a library and extend it with custom importers/exporters or new CLI commands, the TUI will automatically render those new options without requiring modifications to the TUI source code.

## System Design: Dynamic Discovery

Instead of hardcoding forms (e.g., `ImportForm`, `ExportForm`), the TUI will dynamically construct its UI based on the existing metadata of the system.

### 1. Dynamic Menus & Arguments via `argparse`
- The TUI will introspect the `argparse.ArgumentParser` returned by `contracthub.interfaces.cli._build_parser()`.
- **Sidebar Tabs:** Auto-generated based on the sub-parsers available (filtering out `enrich`).
- **Form Inputs:** Auto-generated by inspecting the arguments of each sub-parser (e.g., `required`, `choices`, `type`).

### 2. Extensible Choices via Registries
- For dropdowns like `--format`, the TUI will dynamically pull available options from the internal registries (e.g., `importer_factory.registry.keys()` or dynamically evaluating the `choices` property from argparse which gets populated by the factories).
- This ensures that if a consumer adds a custom plugin (e.g., `@importer_factory.register('their_format')`), it automatically appears in the TUI.

### 3. Execution Bridge
- The UI will gather user inputs and construct a raw arguments list (e.g., `["import", "--format", "sql", "--source", "..."]`).
- It will then pass this list directly to the parser or the specific command executor mapped in `cli.py`, capturing standard output to a Textual Log widget.

## Technology Stack
- **Library:** `textual` (Added to `pyproject.toml`)

## Target User Experience (k9s style)
- Invoked via `contracthub tui`.
- Left panel: List of discovered commands.
- Main panel: A dynamically generated form for the selected command.
- Bottom panel: Output console capturing logs and results.
